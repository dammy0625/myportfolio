An app that uses a third-party API

this ais an app that relies on a third party api to fetch data
and then modify the data to be sent to the user

the api used is called "Bored api"
the app sends requests on-changing of the selection box value
and also on-key up after inputing a number

the app generates random ideas on activities to do when bored
activity based on the option you select or based on the number of participants you put in

what happens in the background??
in the background, the app uses "GET" method with two different end-points
the first end-point is fired using "http request" when the user selects an option in the category and returns an array of objects which the app modifies to give a user an activity
the second endpoint works like the first , but is fired when a user inputs a number value..its response is activities based on the number of paticipants you input
